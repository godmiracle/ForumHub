# ForumHub 项目上下文摘要

本文件是 AI Agent 和新协作者快速理解项目的中文入口。更完整的领域术语、数据源合约和业务不变量请继续阅读 [../CONTEXT.md](../CONTEXT.md)。

## 一、项目目标

ForumHub 是一个多社区 iOS 阅读客户端。项目从 NGA 原型演进而来，目前同时支持 NGA、V2EX 和 LINUX DO，并通过共享领域模型统一列表、详情、账户和本地状态体验。

## 二、核心边界

- 产品层统一使用 `ForumHub` 命名。
- NGA、V2EX、LINUX DO 的差异由各自数据源适配器处理。
- Feature 和 View 使用共享领域模型，不直接解析远端响应。
- 数据源字段、编码、分页、登录和降级规则应保留在 Provider、Repository 或 Parser 层。
- 帖子正文以 `ForumPostDocument` 为唯一权威来源；`body` 是 `normalizedText` 的只读兼容投影，不得覆盖或替代数据源原始内容。
- 收藏、浏览历史、屏蔽用户和频道订阅目前以本地优先为主。

## 三、主要领域概念

- **Forum Source**：远端社区数据源。
- **Forum Channel**：数据源内部可选择的版块、节点或分类。
- **Forum Feed**：频道或热榜对应的分页帖子列表。
- **Topic**：列表中的帖子摘要。
- **Thread Detail**：主楼与回复组成的帖子详情。
- **Forum Provider**：实现 `ThreadRepository` 接缝的数据源模块。
- **Login Session**：各数据源自己的登录凭证和恢复状态。
- **Auth Session Descriptor**：上层账户 UI 使用的统一会话摘要，不承载原始凭证。
- **Favorite Thread**：本地收藏，部分数据源可额外同步远端收藏。
- **Blocked User**：按数据源隔离的本地屏蔽规则。

## 四、关键不变量

- 公共浏览不依赖登录，会话能力按数据源单独管理。
- View 不解析 NGA、V2EX 或 LINUX DO 原始响应。
- 所有持久化身份和帖子标识必须包含数据源维度，避免跨站冲突。
- 帖子详情中的倒序、只看楼主等属于本地展示状态，不应修改 Repository 输出语义。
- 信息流 `summary` 仅是主题摘要；详情请求完成前不得把它渲染为主楼正文，加载失败时应展示明确的加载或错误状态。
- `ForumThread` 的内容相等必须覆盖正文、内容文档、回帖和展示元数据；需要判断帖子身份时显式使用 `source + id`，不得用内容相等代替身份键。
- NGA 连续阅读可以在 UI 层累积远端分页，但底层仍保持数据源分页和去重语义。
- 登录原始 Cookie、Token 和 Keychain 数据不得通过共享描述对象向上层泄露。
- 新的数据源能力应优先通过现有共享协议接入，而不是在 View 中增加大量来源分支。

## 五、当前限制

- iCloud 同步暂未启用，本地数据在卸载或换机时可能丢失。
- V2EX 和 LINUX DO 暂未提供完整的原生回复能力。
- 部分测试目标和真机验证仍受当前 Xcode Beta、签名或依赖状态影响。
- 真机行为优先于模拟器结果，尤其是登录、Cookie、图片、性能和系统权限。

## 六、文档维护

以下内容变化时，应同步更新本文件和 [../CONTEXT.md](../CONTEXT.md)：

- 新增数据源；
- 领域术语或核心不变量变化；
- 登录、分页、持久化或数据源边界变化；
- 上层 UI 与数据源适配层之间的职责变化。
