# 迁移验证记录模板

每个迁移切片完成必要验证后，将以下内容追加到当天的 `YYYY-MM-DD.md`。同一天可以记录多个切片；未实际执行的项目必须写“未执行”并说明原因。

```md
## <迁移编号与名称>

- 日期与时间：<本地时间>
- 变更范围：<文件或模块>
- 设备：<名称、型号、设备标识；不涉及真机时写“不适用”>
- 系统与构建：<OS 版本、Xcode/构建配置>
- 测试命令：`<可复现命令；过长参数可换行>`
- 结果：<通过/失败/未执行，以及测试范围；不复制完整日志>
- 人工验证：<路径与结果，或未执行原因>
- 已知限制：<本次验证未覆盖的内容>
- 回滚点：<可独立撤销的文件或实现接缝，不填写未经创建的 commit>
```

记录规则：

- 自动化日志已包含的逐条测试名称、耗时和编译过程不重复粘贴；
- 真实会话测试必须注明只读边界，禁止记录 Cookie、Token 或账号密钥；
- 性能切片应链接独立基线，不在会话记录中复制整份采样数据；
- 只有验收标准满足且必要验证通过后，才能同步勾选 `docs/todo.md`。
