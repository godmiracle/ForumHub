## Context

首页当前由 `ContentView` 组装 `ForumTopBar` 和 `ForumFeedContent`。来源、搜索、刷新、发帖、频道、当前版面、子版、会话状态、排序和置顶开关横跨多个局部状态；当前频道会重复显示，筛选无法自然扩展，成功登录状态长期占据阅读空间。`ForumViewModel` 只保留 NGA 的 `isAuthenticated` 布尔值和字符串错误，无法让 UI 区分检查中、主动退出、凭证缺失和服务端明确失效。

时间也没有统一契约：NGA Parser 把数字或字符串直接写入 `ForumThread.createdAt/lastReplyAt`，V2EX Mapper 先格式化，Discourse 又有自己的格式。共享列表直接展示字符串，因此原始 Unix 时间戳会泄漏，排序与展示可能对同一个值产生不同理解。

本变更横跨 Feed、Session、Domain 和三个来源适配器，但必须保持既有边界：View 不解析来源响应，Token/Cookie 不进入共享状态，网络或论坛故障不能被伪装成登录失效。

## Goals / Non-Goals

**Goals:**

- 建立稳定、紧凑且可折叠的首页信息层级，减少重复信息并提升帖子可视面积。
- 将排序与筛选分离：排序保持高频互斥控件，低频条件集中在可扩展筛选面板。
- 用类型化会话状态驱动“隐藏 / 登录 / 重新登录”，并支持认证成功后续接发帖意图。
- 在来源接缝解析时间，在共享领域与展示层统一排序及 `MM-dd HH:mm` 输出。
- 让筛选、时间、登录和布局行为可单元测试、可 UI 测试并适配动态字体和 VoiceOver。

**Non-Goals:**

- 不改变 NGA、V2EX 或 LINUX DO 的远端接口和服务端排序语义。
- 不把 Cookie、Token、来源错误码或原始 DTO 暴露给 SwiftUI View。
- 不在首页加入尚未确定的未来筛选业务规则；本次只提供可扩展承载结构。
- 不把普通网络失败、403 权限不足或论坛不可用统一当作登录失效。
- 不凭空假定来源具有新主题提交 API；发帖入口必须经 capability 路由到已验证的来源动作，无法提供有效动作时不得保留无响应按钮。

## Decisions

### 1. 首页采用固定信息顺序和两级滚动头部

完整状态依次为：来源菜单 → 搜索/刷新/发帖 → 频道 → 排序/筛选 → 条件式会话提示 → 帖子列表。删除频道下方重复的 `forum.title` 行。向下浏览后，来源与搜索动作区可折叠，频道以及排序/激活筛选摘要保留为紧凑可发现区域；回到顶部或反向滚动时恢复。

选择该方案是因为高频导航和当前过滤上下文必须随时可见，而搜索、刷新和发帖不需要永久占据高度。备选方案是让整个顶部随列表滚走，虽然实现简单，但长列表中切换频道或清除筛选的成本过高。

### 2. 排序是即时展示状态，筛选使用带草稿的统一 Sheet

`FeedSortMode` 继续作为互斥即时状态，通过两段式控件展示。新增领域无关的 `FeedFilterState`，首批字段包含子版选择和 `showsPinnedThreads`；筛选 Sheet 使用草稿值，提供“重置”和“应用”，只在应用时触发一次必要的 Feed 请求。入口显示生效条件数，置顶以 Toggle 表达状态，避免“显示置顶”究竟是动作还是当前状态的歧义。

排序和置顶可见性按来源记忆，子版按来源与父版键记忆；恢复时必须与当前可用子版求交集。切换来源或父版不得继承不相关 ID。备选方案是继续堆叠 `Menu` 和胶囊按钮，但未来增加筛选后会扩大顶部高度，也难以支持批量应用与统一重置。

### 3. 会话展示使用类型化状态而不是布尔值

共享展示状态定义为 `checking`、`signedOut`、`authenticated`、`expired`，由 Session 层的安全描述对象提供，不携带凭证。凭证缺失或主动退出产生 `signedOut`；Cookie/Token 恢复期间为 `checking`；验证成功或已验证请求成功产生 `authenticated`；只有 HTTP 401、来源明确的无效会话响应、或已保存身份恢复为来源游客身份等可验证证据产生 `expired`。

首页对 `authenticated` 不渲染状态，对 `signedOut` 渲染“登录”，对 `expired` 渲染“重新登录”，`checking` 不闪现错误 CTA。网络超时、离线、限流、5xx、一般 403 保留各自错误语义。备选方案是从 `errorMessage` 字符串推断状态，但这会把文案变化变成业务逻辑并产生误判。

### 4. 发帖意图由协调层续接，入口不得无响应

首页发帖按钮先查询活动来源 capability。已认证且有已验证发帖目的地时立即进入；未登录或失效时记录不含正文和凭证的 `PendingAuthenticatedAction.compose(source, channel)`，完成登录并再次确认来源与频道仍有效后续接；取消登录、切换来源或目的地失效时清除意图。若来源没有已验证的新主题能力，按钮必须隐藏或给出明确“不支持”说明，不能继续使用当前空闭包。

本变更不假设 NGA 的新主题传输格式。实现阶段须先确认项目已有可复用入口或验证来源原生 Web 发帖入口，再把它暴露为 capability；原生提交协议需要独立规格时不得在本次临时猜测。

### 5. 领域时间以结构化值为权威，共享展示层负责格式化

各来源 Adapter/Parser 把 Unix 秒、Unix 毫秒、NGA 日期字符串和 ISO 8601 解析为结构化时间；`ForumThread` 的创建时间与最后回复时间使用可空的结构化值作为排序和展示权威。无法解析时保留诊断但不向列表显示原始技术值。共享 formatter 使用当前时区输出固定 `MM-dd HH:mm`。

列表根据排序模式选择时间：最后回复使用最后回复时间并标注“回复”，最新发帖使用创建时间并标注“发布”；缺少首选时间时可使用另一结构化时间排序，但标签必须反映实际显示字段。选择领域时间而非各 Mapper 输出字符串，可以避免格式、排序和跨年解释在来源间漂移。

### 6. 持久化采用可迁移的展示偏好和时间解码

筛选偏好以版本化、来源/父版作用域键保存，不保存已失效子版。历史和收藏快照中的旧字符串时间在恢复边界尝试解析为结构化值；解析失败时保留条目但不展示原始时间。新写入只写稳定、可解码的时间表示。

### 7. 可访问性与测试是布局契约的一部分

搜索、刷新、发帖、排序、筛选、登录 CTA 必须有稳定 accessibility identifier 和状态化 label。动态字体和窄屏下，频道可横向滚动，操作按钮保持最小点击区域，筛选复杂内容放在 Sheet 中。折叠不得造成 VoiceOver 焦点跳失或重复朗读隐藏控件。

## Risks / Trade-offs

- [结构化时间迁移影响多个初始化器和快照] → 分来源增加解析 Fixture，先建立兼容解码，再切换共享模型，最后删除字符串直显路径。
- [服务端失效信号不一致] → 只接受类型化、来源明确的证据；不确定错误保持原错误分类，并记录测试样本后再扩展映射。
- [滚动折叠与下拉刷新、横向频道手势冲突] → 使用单一纵向滚动状态源，设置滞回阈值，并在真机验证快速滚动、回顶、横滑频道和 refreshable。
- [筛选记忆造成“空列表”困惑] → 始终显示激活数量、空结果原因和一键重置；恢复时清理已不存在的条件。
- [登录完成后上下文已经变化] → Pending action 绑定来源与频道，续接前重新验证，任何不匹配都取消并给出说明。
- [发帖能力范围扩大] → capability 默认关闭，只接入已验证入口；未验证的原生提交协议拆分为后续 change。

## Migration Plan

1. 建立类型化时间与会话状态及其解析/状态转换测试，不先改 UI。
2. 为旧历史、收藏和现有 Fixture 增加兼容解码，确保条目不因迁移丢失。
3. 引入筛选状态、作用域持久化和 Feed 请求批处理，再替换现有分散状态。
4. 重组首页布局，接入条件式会话提示、排序时间语义和可访问性标识。
5. 接入经验证的发帖 capability 与 pending action；没有有效目的地时移除无响应入口。
6. 增加折叠行为，执行聚焦测试、完整 `ForumHubTests` 和已解锁真机验收。
7. 回滚时可恢复旧顶部组合和字符串兼容投影；结构化时间解析与安全会话状态可保留，因为它们不改变远端接口。

## Open Questions

- NGA 当前没有项目内已实现的新主题提交入口；实现前需确认采用经验证的来源 Web 发帖入口，还是另开 change 设计原生提交协议。无论选择哪一种，本次不得交付无响应的发帖按钮。
