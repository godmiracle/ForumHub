## Why

网事杂谈筛选当前把 NGA 全站目录中除主版以外的所有栏目都当作“子版”，既不符合服务端父子关系，也让用户难以定位真正相关的子版。NGA 已在父版主题列表元数据的 `__F/sub_forums` 中提供权威子版结构，因此需要用这一来源重建筛选，并让新增、取消和暂时不可用的子版得到可解释处理。

## What Changes

- 将网事杂谈的可选子版来源从全站栏目目录切换为父版 `fid=-7` 返回的权威 `__F/sub_forums` 元数据，只展示服务端确认的直接、可浏览子版。
- 保持“网事杂谈主版始终包含”，允许用户多选真实子版后将主版与所选子版主题聚合、去重并按现有规则排序和分页。
- 为子版定义区别于普通栏目 ID 的稳定键和浏览目标，明确区分 `fid`、`stid`、过滤 ID 与来源属性，避免混用 NGA 协议字段。
- 保存按来源和父版隔离的子版选择；首次默认只看主版，新子版不自动加入，旧的全站栏目选择仅在被权威目录确认时保留。
- 对完整目录同步进行快照比较：新增子版可标记为“新”，取消子版从选择中清理；目录请求失败或响应不完整时保留上次确认快照，不误判下线。
- 子版加载部分失败时仍展示主版和其他成功子版，并给出局部状态，不把整个信息流判为失败。
- 增加脱敏真实响应 Fixture、解析契约、偏好迁移、目录增删和聚合失败边界测试，并在已连接真机上验证。

## Capabilities

### New Capabilities

- `nga-authoritative-child-forum-directory`: 定义从父版主题元数据识别真实直接子版、映射浏览目标、维护完整快照以及处理新增、取消和不完整同步的行为。
- `wangshi-child-forum-aggregation`: 定义网事杂谈主版始终包含、多选真实子版聚合、偏好恢复迁移、局部失败和筛选界面的用户行为。

### Modified Capabilities

无。

## Impact

- 主要影响 `ForumHub/Data/NGA` 的主题列表元数据解析与 Repository 接缝、`ForumHub/Features/ForumFeed` 的子版状态、聚合请求和筛选 Sheet，以及对应持久化测试。
- 全站 `home/category`、`favorforum/sync` 与 `tagforums` 继续服务栏目目录和订阅，不再作为网事杂谈子版来源。
- 不修改 NGA 账号的服务器端子版屏蔽设置，不调用 `nuke.php` 修改用户偏好，不改变其他来源的栏目与筛选能力。
- 不改变帖子详情、回复、图片上传或认证协议；实现前必须用登录真机取得并脱敏保存一份当前 `__F/sub_forums` 响应形状，不能只依据历史文档猜测位置字段。
