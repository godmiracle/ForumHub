## Context

ForumHub 当前通过 `NGALiveThreadRepository.fetchChannels()` 合并 `favorforum/sync`、`home/category` 和 `home/tagforums`，再由 `ForumViewModel.availableChildChannels` 把当前栏目之外的所有 NGA 栏目暴露为“子版”。这丢失了父子关系，也把收藏、分类和标签栏目混进网事杂谈筛选。用户选择子版后，现有聚合器会依次加载主版和所选栏目并去重；这个“主版 + 多子版”产品语义需要保留，但选择集合必须改为服务端确认的真实直接子版。

项目文档记录 NGA 主题列表的 `/root/__F/sub_forums/*` 包含 `fid/stid`、名称、说明、过滤 ID 和 attributes，但仓库尚无当前登录态真实 Fixture。未登录命令行请求只返回 403，因此字段位置和可浏览判定必须先由真机登录态响应验证，不能把历史文档直接当作生产合约。

## Goals / Non-Goals

**Goals:**

- 让网事杂谈筛选只展示父版元数据确认的真实直接子版。
- 保留网事杂谈主版始终加载，并支持同时聚合多个已选子版。
- 把子版浏览目标、稳定身份和 NGA 服务器过滤 ID 分开建模。
- 在成功完整同步时识别新增、改名和取消；失败或不完整响应不得破坏上次确认快照。
- 迁移现有偏好，只保留权威目录中仍有效的旧选择。
- 让单个子版加载失败成为局部状态，不阻断主版及其他成功内容。

**Non-Goals:**

- 不调用 `nuke.php` 修改 NGA 账号的服务器端子版屏蔽设置。
- 不重写全站栏目订阅、社区页搜索或首页栏目排序。
- 不为 V2EX、LINUX DO 虚构子版能力。
- 不依据名称、fid 范围或本地硬编码猜测父子关系。
- 不在缺少真实响应证据时承诺 `attributes` 阈值、数组位置或 `fid/stid` 请求参数。

## Decisions

### 1. 父版主题元数据是子版唯一权威，全站目录不参与回退

新增来源适配接缝按父版请求子版目录。NGA 实现使用与 App 相同的登录 Cookie 获取网事杂谈主题元数据，并只解析验证过的 `__F/sub_forums` 合约。`home/category`、收藏同步和标签栏目继续用于全站栏目管理，但不得作为子版缺失时的回退。

选择该方案是因为只有父版响应表达父子关系。继续从全站目录做排除、按名称匹配或维护本地白名单都会在板块增删时重新失真。若权威请求失败，界面展示上次确认快照或明确的暂不可用状态，比展示错误列表更可靠。

### 2. 实现前先固定真实响应契约，再编写生产 Parser

首个实施任务是在已登录真机上取得网事杂谈主题列表元数据，删除 Cookie、账号、帖子正文等敏感信息，只保留 `__F/sub_forums` 及解析所需父版字段形成 Fixture。Fixture 必须确认直接子版边界、`fid/stid` 表达、过滤 ID、attributes 和可浏览请求参数。

2026-07-20 已从真机“哥谭之王”的 ForumHub 登录态请求 `thread.php?fid=-7&page=1&lite=xml&__inchst=UTF8`，并将只含父版身份和 `sub_forums` 的脱敏真实形状保存为 `ForumHubTests/Fixtures/nga-wangshi-authoritative-child-forums.xml`。响应确认父版为 `fid=-7`、名称为“网事杂谈”，`sub_forums` 有 31 个直接一层记录，每个记录均为五项位置数组：位置 0 是浏览标识、位置 1 是名称、位置 2 是说明、位置 3 是过滤 ID、位置 4 是 attributes。

普通 `<item>` 记录的位置 0 按 `fid` 浏览；真实对照请求 `fid=570` 返回对应版面，而 `stid=570` 返回错误。`t<ID>` 记录的位置 0 按 `stid` 浏览；真实对照请求 `stid=47206901` 成功，而 `fid=47206901` 返回 404。容器标签中的 `ID` 必须与位置 0 相同，否则结构无效。过滤 ID 不参与浏览请求。当前真实 attributes 出现 `296`、`2590`、`4654`、`4906`，Parser 只保存其原值，不使用 `attributes > 40` 或其他阈值推断目标类型。

若未来响应不能证明这些字段的含义，实施暂停并更新设计，而不是猜测。历史 API 文档只作为调查线索，不是测试预期的唯一来源。

### 3. 领域接口暴露可浏览子版，不向 View 泄露 NGA 原始字段

Repository 接缝返回按父版作用域的子版目录快照。共享表示至少包含父版身份、稳定子版键、展示名称和可用于现有 `fetchForum` 的 `ForumChannel` 浏览目标；`fid/stid` 联合类型、过滤 ID、attributes 和位置数组解析留在 `Data/NGA` 内部。

稳定键必须区分目标种类，例如 `fid:123` 与 `stid:123` 不得碰撞。过滤 ID 仅用于解释和未来来源能力，不能被误传为浏览 fid。本变更不把来源字段继续塞进一个含义不明的 `Int selectedChildChannelIDs`。

### 4. 只有完整成功快照才能确认目录变化

每个父版保存最近一次确认成功的轻量快照。新快照通过完整结构验证后才原子替换，并按稳定键计算：

- 当前有、旧快照无：新增，默认未选择并显示“新”；
- 两边都有、名称变化：更新名称并保留选择；
- 旧快照有、当前无：取消，从有效选择中移除；
- 请求、解码或结构验证失败：保留旧快照，不产生取消事件。

“新”状态保留到用户打开筛选并完成一次关闭或应用，避免后台刷新后标记瞬间消失。没有旧快照时，首次成功结果建立基线，不把所有既有子版标成新增。

### 5. 筛选明确表达“主版始终包含，子版可多选”

筛选 Sheet 的子版区只显示权威直接子版。网事杂谈主版以不可取消的说明行呈现；“只看主版”清空子版草稿；多项切换后仍通过“应用”一次提交和刷新。默认选择为空，新子版不自动加入。

如果真实子版数量较少，保持直接列表；数量超过可用性阈值时再显示本地名称/稳定键搜索。筛选入口可以显示已选数量和未确认新增数量，但不得把全站栏目数量计入。

### 6. 聚合以主版成功为最低可用条件，子版失败局部降级

聚合请求继续使用现有请求 generation、取消、分页和 `source + tid` 去重接缝。主版请求失败时整个 Feed 失败；主版成功后，单个子版失败只记录该子版的局部失败并合并其他成功结果，界面显示简短提示。当前先保持顺序请求，避免无测量依据地对 NGA 形成突发并发；若真机数据证明延迟不可接受，再独立评估有限并发。

每一远端页仍按相同页码请求主版和已选子版，合并后沿用当前“最后回复 / 最新发帖”排序。新增目录同步不得与正在进行的 Feed generation 交叉提交旧状态。

### 7. 偏好升级延迟到权威快照可用时完成

子版偏好升级为稳定键集合，继续按 `source + parent channel` 隔离。旧 `Int` 选择只有在首次权威快照中能唯一映射为真实子版浏览目标时才保留；全站其他栏目被丢弃。若首次目录同步失败，保留旧记录但不用于发起未经确认的子版请求，等待成功快照后再迁移并持久化。

排序与置顶的现有来源级偏好不在本变更中改动。

## Risks / Trade-offs

- [真实 `sub_forums` 合约与历史文档不同] → 先捕获脱敏真机 Fixture；字段无法确认时停止实现并修订设计。
- [目录接口需要有效登录且可能暂时失败] → 使用最近一次确认快照；没有快照时显示不可用，不回退到全站栏目。
- [父版可能已经包含部分子版主题，额外请求会产生重复] → 保留 `source + tid` 去重，并用真实 Fixture/真机验证聚合结果与排序。
- [多个子版顺序请求增加首屏时间] → 先保持服务器友好的现状并测量；有证据后再评估有上限的并发。
- [旧偏好中的 Int 无法无歧义表达 stid] → 只迁移能由权威快照唯一确认的条目，其余安全丢弃。
- [一次完整响应误判取消会删除选择] → Parser 必须验证父版身份和完整子版容器；任何结构疑点都按失败处理，不提交快照。

## Migration Plan

1. 从登录真机捕获并脱敏网事杂谈子版元数据 Fixture，验证浏览目标与直接子版边界。
2. 增加 NGA 子版 Parser、稳定身份和 Repository 接缝，保持现有全站栏目路径不变。
3. 增加权威快照与偏好迁移，让旧选择只在成功目录后归一化。
4. 将筛选 Sheet 改为主版固定、多真实子版草稿选择，并接入新增/取消状态。
5. 调整聚合器支持局部失败，运行 Parser、偏好、generation、分页和去重测试。
6. 在“哥谭之王”真机验证目录刷新、多选聚合、新增/取消模拟、前后台恢复和失败降级。

回滚时可以恢复旧筛选入口并停止读取新偏好，但不得重新宣称全站栏目是真实子版；若权威目录尚不稳定，安全回滚形态是暂时隐藏子版选择、只加载网事杂谈主版。
