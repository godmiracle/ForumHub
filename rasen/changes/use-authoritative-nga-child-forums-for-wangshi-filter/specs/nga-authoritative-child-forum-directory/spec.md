## ADDED Requirements

### Requirement: 子版目录使用父版权威元数据
系统 SHALL 仅从指定 NGA 父版主题元数据中已验证的子版容器建立真实直接子版目录，不得使用全站分类、收藏或标签栏目推断父子关系。

#### Scenario: 网事杂谈返回有效子版元数据
- **WHEN** 登录态主题元数据确认父版为网事杂谈并包含结构完整的子版容器
- **THEN** 系统只返回该容器中满足已验证可浏览合约的直接子版

#### Scenario: 权威目录不可用
- **WHEN** 父版子版请求失败、解码失败或结构无法通过完整性验证
- **THEN** 系统不得用全站栏目代替真实子版，并保留最近一次确认成功的快照或显示目录暂不可用

### Requirement: 子版身份区分浏览目标与来源过滤字段
系统 MUST 为子版保存可区分 `fid` 与 `stid` 的稳定身份和浏览目标，并将 NGA 过滤 ID、attributes 等来源字段限制在适配层内，不得把过滤 ID 当作浏览 fid。

#### Scenario: 数值相同但目标种类不同
- **WHEN** 两个子版目标分别为 `fid:123` 与 `stid:123`
- **THEN** 系统将它们视为不同稳定身份，并使用各自经过 Fixture 验证的请求参数

#### Scenario: 响应包含过滤 ID
- **WHEN** 子版条目同时包含浏览目标和服务器端过滤 ID
- **THEN** 系统只用浏览目标加载主题，且本变更不调用服务器端屏蔽接口

### Requirement: 目录契约由脱敏真实 Fixture 保护
系统 MUST 在生产解析前以当前登录真机响应建立脱敏 Fixture，覆盖直接子版边界、字段位置、目标种类和可浏览判定；无法由 Fixture 确认的字段不得凭历史文档补全。

#### Scenario: 真实响应与历史文档一致
- **WHEN** 脱敏 Fixture 证明字段含义和结构边界
- **THEN** Parser 以该响应形状建立契约测试并只输出已确认字段

#### Scenario: 真实响应无法确认关键字段
- **WHEN** Fixture 不能确定浏览目标或直接子版边界
- **THEN** 实施停止并更新设计，不发布猜测性 Parser

### Requirement: 完整快照驱动新增与取消
系统 SHALL 仅在新的权威目录通过完整验证后原子替换父版快照，并按稳定身份识别新增、改名和取消；失败或不完整同步不得产生取消事件。

#### Scenario: 服务端新增直接子版
- **WHEN** 完整新快照包含旧基线中不存在的稳定子版键
- **THEN** 系统将其标记为新增但保持未选择，直到用户明确选择

#### Scenario: 服务端取消已选子版
- **WHEN** 完整新快照不再包含旧快照中的已选稳定键
- **THEN** 系统从有效选择中移除该键并提供一次可理解的变更提示

#### Scenario: 同步只返回部分或异常结构
- **WHEN** 任一权威完整性条件未满足
- **THEN** 系统保留旧快照和选择，不把缺失条目判定为取消

#### Scenario: 子版改名
- **WHEN** 完整新快照保留稳定键但改变展示名称
- **THEN** 系统更新名称并保留用户选择
