## Context

栏目页当前直接消费 `ForumViewModel.channels`。NGA 的该数组由 `favorforum/sync`、`home/category` 和 `home/tagforums` 合并得到，能够服务全站普通栏目，但不表达父子关系，也不保证包含网事杂谈 `__F/sub_forums` 中以 `stid` 浏览的直接子版。栏目搜索只是对这份扁平数组匹配标题和 `nativeKey`，因此真实 Fixture 中的 `[股市]技术分析`（`stid:47206901`）不会出现。

网事杂谈已经有另一条经过真机 Fixture 验证的权威目录链路：Repository 严格解析父版元数据，`AuthoritativeChildForumDirectoryStore` 保存最近完整快照，Feed 只用稳定 `fid:/stid:` 键恢复筛选并处理新增、改名、取消和失败保留。该链路目前仅在选中网事杂谈主版时投影到筛选 UI，没有成为栏目管理的数据源。

现有订阅以 `source + ForumChannel.nativeKey` 保存；普通 NGA 栏目通常是裸数字键，而权威子版使用 `fid:<id>` 或 `stid:<id>`。如果直接拼接两个数组，同一 `fid` 可能出现两次，`ForumChannel.id` 的裸 `Int` 也不足以区分同值的 `fid/stid`。本变更需要同时处理目录组合、导航身份、订阅迁移和 UI 信息层级，不能只修搜索过滤函数。

## Goals / Non-Goals

**Goals:**

- 让栏目页同时发现普通栏目和最近确认的网事杂谈权威直接子版。
- 允许权威子版作为首页顶部独立栏目订阅、排序和浏览，并保持正确的 `fid/stid` 请求语义。
- 让首页独立订阅与网事杂谈聚合筛选保持两套互不联动的偏好。
- 以规范化稳定身份去重、导航和持久化，并无损迁移现有有效订阅顺序。
- 重写栏目页层级，清晰分离“已添加到首页”“浏览全部栏目”“搜索结果”和“排序编辑”。
- 复用权威快照的 fail-closed 规则处理新增、改名、取消和失败。

**Non-Goals:**

- 不从全站栏目接口推断父子关系，也不按名称、ID 范围或 attributes 猜测子版。
- 不改变网事杂谈“主版始终包含”的聚合语义，不把首页订阅同步成筛选选择。
- 不调用 `nuke.php`，不修改 NGA 服务器端收藏、屏蔽或栏目偏好。
- 不为 V2EX、LINUX DO 虚构父子目录；其现有普通栏目订阅继续工作。
- 不重构帖子详情、回复、登录、图片或通用网络层。
- 不引入新的第三方依赖。

## Decisions

### 1. 建立来源无关的栏目目录展示快照，不把权威子版直接追加到 `ForumViewModel.channels`

Community 功能在现有 `ForumChannel` 浏览目标之上建立只读目录展示模型。每个展示项至少携带规范化稳定身份、可浏览 `ForumChannel`、展示标题、可选父版标题、来源种类和订阅状态；目录按当前来源组合：

- 普通栏目来自现有 `fetchChannels()`；
- NGA 网事杂谈直接子版来自最近确认的 `AuthoritativeChildForumDirectory`；
- 同一规范化浏览身份同时出现时只保留一个展示项，权威目录提供父版关系和当前标题；
- V2EX、LINUX DO 没有权威子版时只显示现有普通栏目。

该模型属于 Feature/Presentation 边界，不解析 NGA 原始响应。选择这一方案而不是拼接 `[ForumChannel]`，是为了显式表达层级和来源、避免 `Int id` 碰撞，并让搜索与分组不污染 Repository 输出语义。

### 2. 用规范化导航身份统一列表、选择、订阅和排序

所有栏目管理与首页导航接缝使用同一规范化字符串身份：

- NGA 普通裸数字目标规范化为 `nga:fid:<value>`；
- NGA 权威 `fid:<value>` 规范化为 `nga:fid:<value>`；
- NGA 权威 `stid:<value>` 规范化为 `nga:stid:<value>`；
- 其他来源继续使用 `source:nativeKey`。

远端数字 `ForumChannel.id` 仍可作为请求载荷和展示辅助值，但不得再单独承担栏目列表身份、当前选择或持久化身份。Feed 顶部 `ForEach`、当前栏目高亮和 Community 行身份必须显式使用规范化键，从而允许数值相同但种类不同的 NGA 目标共存。

选择该方案而不是把 `stid` 编码为特殊负数，是因为后者会制造新的隐式协议且不能证明无碰撞；也不把来源字段暴露给 View，View 只消费规范化身份与浏览目标。

### 3. 订阅持久化升级为新版本，并保留旧快照供二进制回滚

新增版本化订阅与排序存储键。首次没有新版本时，从现有 `nga:<裸数字>`、V2EX 和 LINUX DO 键迁移：

- NGA 裸数字按现有合约迁移为 `nga:fid:<value>`；
- 其他来源保持原生键；
- 按规范化键去重并保留已有相对顺序；
- 无法解析的记录丢弃，但不得导致该来源全部有效默认栏目消失；
- 迁移后写新版本，不覆盖旧版本键，以便回滚旧二进制时仍能恢复原有普通栏目。

新订阅的 `stid` 只存在于新版本；旧二进制回滚后看不到这些新增子版，但原有普通栏目不应丢失。至少保留一个当前来源栏目和默认订阅规则继续沿用。

### 4. 权威目录生命周期独立于当前选中的 Feed 栏目

栏目发现不能依赖用户此刻正在浏览网事杂谈主版。Content/ViewModel 通过显式父版 `ForumChannel.defaultForum` 恢复缓存并刷新 NGA 权威目录，沿用启动、切换来源和前台激活接缝；目录请求与普通栏目请求可以各自失败，但组合展示只在各自结果确定后更新对应部分。

失败行为继续 fail closed：

- 有缓存时保留最近确认的子版及其订阅；
- 无缓存时只显示普通栏目，并在 NGA 子版分组展示“暂时无法更新”，不伪造空目录或取消事件；
- 只有完整新快照可以产生新增、改名和取消；
- 新增默认不订阅；改名按稳定键保留订阅；确认取消后清理新版本订阅与排序并产生一次性栏目通知。

目录同步结果需要让筛选偏好和栏目订阅分别消费同一次完整变化，而不是由某一消费者提前销毁取消信息。两类偏好各自归一化，互不写入。

### 5. 子版独立导航复用现有 `ForumChannel` 浏览目标，不进入聚合请求

从首页点击已订阅权威子版时，按该目录项携带的 `ForumChannel.nativeKey` 进入普通单栏目 Feed：`fid:` 使用 API 路径，`stid:` 使用现有 Web forum 路径。此时不加载网事杂谈主版，不读取网事杂谈筛选集合，也不显示“主版始终包含”的聚合状态。

从网事杂谈筛选选择同一子版时，仍由现有聚合器加载“主版 + 已选子版”。订阅开关只改变首页可见栏目；筛选 Apply 只改变父版作用域的聚合偏好。两条路径共享浏览目标与稳定身份，但不共享选择状态。

### 6. 栏目页使用任务导向的信息层级

页面保持当前来源作用域，重组为：

1. 标题与当前来源说明；
2. 吸顶或持续可发现的统一搜索；
3. “已添加到首页”有序列表，默认不显示拖动手柄，进入“编辑排序”后才允许拖动；
4. “浏览全部栏目”，NGA 至少区分“网事杂谈子版”和“其他 NGA 栏目”；其他来源沿用单一普通栏目组；
5. 搜索时用单一结果列表替代完整分组，匹配标题、稳定编号和可用说明，并为子版显示“网事杂谈 › 子版”上下文；
6. 行主体负责打开栏目，独立的添加/已添加控件负责订阅，避免点击语义混淆。

页面使用动态字体、最小点击区域、明确 VoiceOver 标签和值，并为加载、缓存目录、无结果、无确认子版目录和取消通知提供可访问状态。视觉继续使用现有 `PaperTheme`/DesignSystem，不引入另一套颜色或玻璃组件。

### 7. 测试以身份和状态转换为主，真机验证交互层级

自动化覆盖目录去重、裸数字迁移、`fid/stid` 区分、订阅顺序、搜索、直接导航、筛选隔离、完整取消与失败保留。UI/真机覆盖添加 `[股市]技术分析` 后出现在首页顶部、重启恢复、排序、独立浏览、搜索面包屑、较大动态字体和 VoiceOver。

不得用在线响应变化作为唯一自动化前提；继续使用现有脱敏权威目录 Fixture 和可控 Repository/Store 测试。

## Risks / Trade-offs

- [普通目录和权威目录对同一 fid 使用不同键] → 在组合前统一规范化为 `nga:fid:<value>`，并以规范化键测试去重。
- [`ForumChannel.id` 仍是 Int，SwiftUI 行可能错误复用] → 列表、选择和导航显式使用规范化字符串身份，不依赖默认 `Identifiable`。
- [取消事件同时影响筛选和首页订阅] → 让一次完整同步产生可重复消费的变化结果，两类 Store 分别归一化，失败同步不产生删除。
- [新存储格式导致旧版本回滚看不到 stid] → 保留旧普通栏目快照；接受旧二进制无法理解新能力，但保证旧订阅不丢失。
- [独立子版和父版聚合提供两条入口，用户可能混淆] → 用“已添加到首页”和“网事杂谈筛选”明确区分，并在搜索结果显示父版上下文。
- [目录刷新与 Feed 加载交错] → 复用现有 generation/取消边界；目录只改变目录与有效订阅，不允许旧 Feed 结果覆盖新的当前栏目。
- [栏目页重写扩大可访问性回归面] → 保留 DesignSystem，增加聚焦呈现测试并在真机验证动态字体、VoiceOver、拖动与独立按钮点击区域。

## Migration Plan

1. 固定规范化身份和旧订阅迁移测试，再引入新版本存储，保留旧键不覆盖。
2. 建立栏目目录展示快照与组合测试，先让 Community 搜索可发现普通栏目和缓存权威子版。
3. 接入子版订阅、首页可见栏目和规范化选择身份，验证 `stid` 独立导航。
4. 接入完整目录新增、改名、取消和失败保留，使订阅与筛选分别消费变化。
5. 重写栏目页布局、编辑排序、搜索结果和状态反馈，并完成可访问性标识。
6. 运行聚焦测试、完整测试和 Debug 真机构建；在已连接真机完成 `[股市]技术分析` 添加、重启、排序、独立浏览及网事杂谈筛选隔离验收。
7. 验收完成后更新模块文档、领域上下文、决策、待办和 changelog。

回滚时恢复旧栏目页与旧订阅读取路径；由于旧存储键仍保留，普通栏目与原顺序可以恢复。回滚版本忽略新 `stid` 订阅，不得把它误解为 `fid`。

## Open Questions

- 无阻塞问题。产品已确认：从栏目页添加权威子版后，必须作为首页顶部独立栏目出现；该订阅不改变网事杂谈聚合筛选。
